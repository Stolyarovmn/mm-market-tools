# Owner UI Feedback 2026-04-10

Источник: расшифровка разговора с управляющим магазина по текущему dashboard UI.

## Итог

Основной сигнал не про “красоту”, а про ежедневную применимость:

- интерфейс должен объяснять термины;
- подсказки не должны ломаться;
- SKU должен быть виднее названия;
- жизненный цикл действия должен быть прозрачен;
- dashboard должен читатьcя как рабочий инструмент, а не как набор исследовательских блоков.

## Разногласия

Явных методологических разногласий с этим feedback нет.

Есть только инженерная оговорка:

- пункт 9 про auto-lifecycle задач нельзя закрывать “косметикой”;
- там нужен отдельный формальный workflow, а не просто новые подписи в UI.

Поэтому замечание принято, но реализовываться должно как отдельный product/operational слой.
